# stm32-mcp

STM32L475 (B-L475E-IOT01A) firmware for the metalbot project.
Receives steering and throttle commands over BLE and drives PWM outputs to control servos/ESC directly.

## Architecture & Version

- **Target MCU**: STM32L475 (Cortex-M4 with FPU)
- **Board**: B-L475E-IOT01A (Discovery Kit IoT Node)
- **Current Version**: 0.2.0-ble
- **Framework**: STM32Cube HAL + BlueNRG-MS BLE middleware
- **BLE Module**: SPBTLE-RF (BlueNRG-MS) via SPI3

## Features

- **BLE GATT Control Service** (`0xFE40`)
  - **Command Characteristic** (`0xFE41`): Write — receives `[int16_t steering, int16_t throttle]` pulse widths in µs
  - **Status Characteristic** (`0xFE42`): Notify — firmware heartbeat/status (reserved)
- **PWM Servo Output** via TIM3
  - `PB1` (TIM3_CH4) → Steering servo
  - `PB4` (TIM3_CH1) → Throttle ESC
- **Safety Watchdog**: 1.5s timeout reverts to neutral (1500µs) on connection loss
- **Device Name**: `METALBOT-MCP`

## Build and Flash

A convenience script `build.sh` is provided in the root of the project to build and flash the firmware using STM32CubeCLT.
Make sure you have STM32CubeCLT installed at `/opt/st/STM32CubeCLT_1.21.0/` or set `CUBECLT_ROOT` correctly in your environment.

### Quick Commands

- **Build Debug firmware:**
  `./build.sh`
- **Build Release firmware:**
  `./build.sh -r build`
- **Flash firmware over SWD (with ST-Link):**
  `./build.sh flash`
- **Clean builds:**
  `./build.sh clean`

For full details, run `./build.sh --help` or refer to the `.agent/workflows/build-stm32.md` workflow.

## Directory Structure

- `Core/`: Application source code (main.c, ble_app.c, interrupt handlers)
- `BLE/`: BlueNRG-MS middleware (transport layer, GATT, GAP, timer server, SPI HW)
- `Drivers/`: STM32 HAL and CMSIS libraries (Git LFS)
- `build/`: Compilation output artifacts
- `cmake/`: CMake configuration generated by STM32CubeMX
- `docs/`: Board user manual and datasheets
